JTML

Build small internal tools, prototypes, demos, and teaching apps with a C++ CLI, readable source files, deterministic local builds, and browser runtime behavior that stays close to HTML.

Syntax

Shorter than HTML, still HTML-shaped.

JTML keeps tags, attributes, nesting, events, state, and derived values. Friendly JTML 2 removes closing-tag repetition, statement terminators, and framework ceremony.

jtml 2 selects Friendly syntax page opens the app root click wires actions to events get keeps values reactive

Enterprise posture

Small enough to audit, complete enough to operate.

JTML is not trying to replace your design system or deployment platform. It gives teams a focused local toolchain for HTML-shaped reactive pages, with clear build artifacts and predictable operational boundaries.

Auditable Source

Readable .jtml files compile to regular HTML, and the CLI commands are scriptable in CI.

Local Studio

jtml demo opens Studio with Run, Lint, Fix, Format, Export HTML, diagnostics, compiler artifacts, autosaved drafts, versioned local Save/history, command palette, and live preview.

Two Deployment Modes

Emit static HTML for simple pages or run the tiny live runtime for server-backed events.

Verification Loop

Doctor checks toolkit shape; tests cover lexer, parser, friendly syntax, formatter, linter, and examples.

Try it locally

A five-minute language loop.

Build the CLI, generate a starter page, serve it, edit it, and reload. The language stays close to the browser instead of hiding it.

1

Open Studio

./build/jtml demo --port 8000
2

Create And Check

./build/jtml new my_page.jtml
./build/jtml check my_page.jtml
./build/jtml lint my_page.jtml
3

Serve

./build/jtml serve my_page.jtml --port 8000 --watch
