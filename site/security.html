Toolchain

Security and operations

Built for local control and auditability.

JTML keeps the authoring loop local, emits inspectable artifacts, and separates static publishing from live runtime deployment.

Readiness checklist

What enterprise teams need before trust.

Local-First Tooling

No hosted compiler or SaaS account is required. Studio, tutorial, checker, formatter, linter, examples, and packager run on the developer machine.

Artifact Verification

scripts/verify_all.sh runs unit tests, example smoke tests, static site generation, release packaging, and artifact existence checks.

Static By Default

jtml build writes plain HTML for static hosting. Live runtime is explicit and should sit behind your normal TLS reverse proxy.

Release Evidence

The release script creates a manifest and SHA-256 checksums so downstream packaging systems can record exactly what was shipped.

Operational Boundary

The live server exposes HTTP, WebSocket, and /api/event fallback endpoints. Use --watch only for local development.

Readable Source

JTML files stay close to HTML, which makes review, diffs, and policy scanning simpler than opaque generated frontend bundles.
