Get started

Local toolkit

A complete local product loop.

JTML ships as a local CLI with Studio, runner, tutorial IDE, checker, formatter, linter, smoke tester, VS Code support, static site artifact builder, release packager, and predeploy verification.

Commands

One binary, several loops.

Runner

Serve a JTML file with HTTP, WebSocket updates, and event fallback.

jtml serve examples/friendly_counter.jtml --port 8000 --watch

Tutorial IDE

Launch guided lessons with editable code and live preview.

jtml tutorial --port 8000

Studio

Open the native Studio with Run, Lint, Fix, Format, Export HTML, diagnostics, compiler artifacts, command palette, autosaved drafts, versioned local Save/history, and live preview.

jtml demo --port 8000

Tester

Parse, lint, transpile, and smoke-test every bundled example.

jtml test

Doctor

Check the local toolkit folders, scripts, tutorial, site, editor assets, and emit JSON for CI.

jtml doctor --json

Static Build

Compile a file or app folder to a distributable static HTML output.

jtml build examples/friendly_import_page.jtml --out dist/app

Release Kit

Create the predeploy site artifact and local CLI release archive.

scripts/verify_all.sh

Install

Install the CLI into a prefix for team machines or build images.

cmake --install build --prefix ~/.local

Package

Generate a local release directory, archive, manifest, and checksums.

scripts/package_cli.sh dist/release

Predeploy

Build everything locally.

These scripts stop one step before publishing a public domain. They produce local artifacts under dist/.
